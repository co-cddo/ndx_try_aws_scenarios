## Project Overview

NDX:Try AWS Scenarios is a zero-cost AWS evaluation platform for UK local government. It consists of a documentation portal (Eleventy/GOV.UK) and 6 deployable AWS CloudFormation mini-applications (Council Chatbot, Planning AI, FOI Redaction, Smart Car Park, Text-to-Speech, QuickSight Dashboard). Councils can deploy real AI/cloud scenarios in 15 minutes and generate committee-ready evidence packs.

**Part of the [LGA AI Hub](https://www.local.gov.uk/our-support/cyber-digital-and-technology/artificial-intelligence) ecosystem.**

## Project Structure

This project consists of 2 parts:

### Documentation Portal (portal)

- **Type:** web
- **Location:** `src/`
- **Tech Stack:** Eleventy 3.x, GOV.UK Frontend 5.13, Nunjucks, YAML data
- **Entry Point:** `eleventy.config.js`
- **Purpose:** Static website documenting 6 AWS scenarios with walkthroughs, evidence packs, and deployment guides

### AWS Scenario Templates (aws-scenarios)

- **Type:** infra
- **Location:** `cloudformation/`
- **Tech Stack:** AWS CloudFormation/SAM, Python 3.12, Lambda, Bedrock, S3
- **Entry Point:** `cloudformation/scenarios/*/template.yaml`
- **Purpose:** 6 deployable serverless applications demonstrating AWS services for UK councils

## Cross-Part Integration

The portal serves as the user-facing documentation and deployment interface for the AWS scenarios:

1. **Scenario Data** (`src/_data/scenarios.yaml`) defines all 6 scenarios with metadata, deployment URLs, and CloudFormation parameters
2. **Deploy URLs** are generated by Eleventy filters pointing to CloudFormation Quick Create Console URLs
3. **Screenshot Pipeline** captures live screenshots from deployed AWS scenarios for documentation
4. **Evidence Packs** are generated from templates using scenario-specific data and metrics

## Getting Started

### Portal Setup

**Prerequisites:** Node.js 20+

**Install & Run:**

```bash
# Install dependencies
npm install

# Start development server
npm start
# Open http://localhost:8080

# Build for production
npm run build
```

### AWS Scenarios Setup

**Prerequisites:** AWS account with appropriate permissions

**Deploy a Scenario:**

1. Visit any scenario page on the portal
2. Click "Deploy to AWS" button
3. CloudFormation Quick Create opens in AWS Console
4. Accept defaults and create stack (3-8 minutes)
5. Access deployed application via stack outputs
